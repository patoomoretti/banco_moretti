# Proyecto Personal – Sistema Bancario (Backend)

## 📌 Descripción
Este proyecto es un **sistema bancario desarrollado como proyecto personal**, enfocado en la **implementación de la lógica de negocio backend** utilizando **Python sin frameworks**.

El objetivo principal es **comprender y modelar el funcionamiento interno de un sistema bancario**, priorizando la lógica, las validaciones y el flujo de datos antes de incorporar frameworks, bases de datos o interfaces gráficas.

---

## 🎯 Objetivo del proyecto
- Practicar pensamiento backend y lógica de negocio
- Modelar operaciones bancarias reales
- Trabajar con estructuras de datos en memoria
- Preparar una base sólida para futuras etapas (API, persistencia, framework)

Este proyecto forma parte de un proceso de aprendizaje progresivo, similar al que se utiliza en entornos reales de desarrollo.

---

## ⚙️ Funcionalidades implementadas
- Alta y gestión de clientes
- Creación y administración de cuentas bancarias
- Consulta de saldo
- Depósitos y retiros
- Validaciones de operaciones
- Control de errores lógicos (fondos insuficientes, datos inválidos, etc.)

> Todas las operaciones se realizan utilizando **listas, diccionarios y estructuras básicas**, simulando el comportamiento interno de un backend.

---

## 🧠 Enfoque técnico
El proyecto fue desarrollado **sin frameworks** de forma intencional para:
- Comprender el flujo completo de una operación
- Separar responsabilidades lógicas
- Evitar depender de herramientas externas en la etapa inicial
- Facilitar una futura migración a un framework como Flask

---

## 🛠️ Tecnologías utilizadas
- Python
- Estructuras de datos nativas (listas, diccionarios)
- Programación modular
- Lógica orientada a procesos
